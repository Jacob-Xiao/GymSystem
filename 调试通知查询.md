# 调试通知查询问题

如果通知没有显示，可以按以下步骤调试：

## 1. 检查数据库中的通知数据

在MySQL中执行：
```sql
USE gym_management_system;
SELECT * FROM notification;
```

检查：
- 是否有通知数据
- target_type 的值是 'all' 还是 'specific'
- target_accounts 的内容（如果是specific类型）

## 2. 测试API接口

在浏览器中访问（替换account为实际的会员账号）：
```
http://localhost:3001/api/notification/member/202100788
```

查看返回的JSON数据。

## 3. 检查浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 切换到"网络"（Network）标签
3. 刷新页面或点击通知标签
4. 查看 `/api/notification/member/xxxxx` 请求
5. 检查响应内容

## 4. 常见问题

### 问题1：通知类型是'all'但不显示
- 检查后端服务是否正常运行
- 检查API路由是否正确注册

### 问题2：通知类型是'specific'但不显示
- 检查target_accounts JSON数组中是否包含该会员账号
- 检查账号类型是否匹配（整数 vs 字符串）

### 问题3：API返回空数组
- 检查数据库中是否有通知数据
- 检查通知的target_type和target_accounts是否正确
